## 二、各说明书统一内容模板

每人写自己的模块说明书时，**严格按以下结构**（共 8 节），保持格式一致，方便组长后期整合。

---

### 2.1 绪论（约200字）
- 项目背景简述：自由职业平台的市场需求、Freelite 的定位
- 本模块解决的问题、在整体系统中的角色
- 说明书的组织结构

### 2.2 需求分析（约400字）
- **功能需求**：本模块需要实现的核心功能列表（用编号列出）
- **非功能需求**：响应速度、安全性（如 JWT 鉴权）、并发处理等
- **用例描述**：每个功能对应的用户操作场景

### 2.3 数据库设计
- **涉及的表**：列出模块用到的全部表，每张表给出字段名、类型、是否主键/外键、约束
- **关系说明**：表与表之间的外键关联（文字描述即可）
- **ER 图**：画本模块的实体关系图（**手绘拍照** 或 **draw.io / Excalidraw 电子版**）
- **关键 SQL**：核心增删改查 SQL 语句各 1-2 条示例

### 2.4 功能模块图
- 画出本模块的功能拆解结构图（**树状图**），展示：子功能 → 子功能的子功能
- 示例：用户系统 → 注册功能（表单校验、重复检测、密码加密） / 登录功能（JWT生成、会话保持）

### 2.5 核心业务流程 + 流程图
- 梳理本模块 2-3 个核心流程，用**步骤序列**描述
- 每个流程配上 **流程图**（手绘拍照 或 电子版）
- 示例流程（竞标模块）：
  ```
  流程：用户竞标项目
  1. 用户浏览项目列表 → 点击项目标题进入详情
  2. 填写竞标金额 + 留言 → 点击"提交竞标"
  3. PlaceBidServlet 接收参数 → 校验用户是否已登录、项目是否存在
  4. BidDao.insertBid() 将竞标记录写入 bids 表
  5. 成功 → 跳转到项目详情页显示"竞标成功"提示
  ```
- 关键判断条件用分支符号标注（菱形）

### 2.6 界面设计与核心代码实现
- **页面截图**：每个主要功能的 JSP 页面截图（1-2 张），说明页面布局、表单字段、按钮功能
- **核心代码片段**：选 3-5 段关键 Java 代码（Servlet 逻辑 / DAO 方法 / 实体类），每段后写注释说明做了什么
- **代码排版要求**：使用等宽字体、三线表展示代码（参考组要求的三线表格式，可用 Markdown 表格模拟）

### 2.7 测试用例
- 用 **三线表** 格式列出测试用例

| 编号 | 测试项 | 测试步骤 | 预期结果 | 结果 |
|---|---|---|---|---|
| TC-A-1 | 正常注册 | 填写合法信息 → 提交 | 注册成功，重定向到登录页 | ✅ |
| TC-A-2 | 用户名重复 | 使用已存在的用户名注册 | 提示"用户名已存在" | ✅ |
| TC-A-3 | 未登录访问个人资料 | 直接访问 profile.jsp | 重定向到登录页 | ✅ |

- 至少 5 个测试用例（含正常路径 + 错误处理）

### 2.8 功能总结与致谢
- 本模块实现了什么功能、技术亮点、遇到的困难和解决方案
- 致谢：指导老师、团队成员、开源社区等

## 四、整合版说明书法（组长陈怡安）

整合版在四份个人说明书完成后编写，内容包括：

## 六、常见问题

**Q: 流程图/ER 图用什么画？**
A: 推荐 draw.io（在线免费）、Excalidraw、ProcessOn，或者手绘拍照。不接受 Word 自带的形状图（太丑）。

**Q: 三线表什么样？**
A: 即学术论文常见的表格格式：只有顶线、表头底线、底线三条横线，中间无竖线。例：

| 字段名 | 类型 | 说明 |
|---|---|---|
| id | INT(11) | 主键 |
| username | VARCHAR(50) | 用户名 |

（这就是三线表头样式，正文里所有表格统一）

**Q: 代码写在哪？**
A: 写在 "界面设计与核心代码实现" 这一节，用 Markdown 代码块（```java … ```）展示，每个代码段前写文字说明。
